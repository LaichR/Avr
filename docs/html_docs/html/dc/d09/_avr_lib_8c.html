The default message handler handles all read and write regiseter requests from the PC. Custom implementations can call this DefaultHandler. In case the return value is False, the Default handler did not do anything and it is appropriate to do something with the message.

◆ RegisterCompareMatchInterrupt()

void RegisterCompareMatchInterrupt ( CompareMatchSource  source,
TimerFrequency  frequency,
uint8_t  match,
IsrHandler  handler 
)

Setup compare match interrupt on one of four available interrupt sources.

This function sets up the timer block in order to generate compare match interrupts Note: CompareMatchSource0 and CompareMatchSource1 refer to the HW Tcnt0 CompareMatchSource2 and CompareMatchSource3 refer to Tcnt1. Changing the frequency changes the frequenty of both interrupt sources of the timer! Apparently CompareMatch1 only is operational if CompareMatch0 is running and CompareMatch3 is only operational if Comparematch2 is running. This seems to be a HW constraint.

Parameters
sourcesource of comparematch interrupt
frequencydivision factor for timer. The base frequency is 16Mhz
handlerthe function that does the work

◆ StartTimer()

void StartTimer ( void  )

Start the Timer on Tcnt1 and reset internal counters.

Tcnt1 holds a 16bit counter. In case the counter overflows, a int32 counter variable is incremented. The counter variable is supposed to refelct the us since the last start of the counter! (Timer-Frequency is set to 2Mhz) The timer variable will hence overflow every 2^32 us 4295s > 1h
